# 注意事项

## 立绘命名规则

立绘文件名由“服装 ID”和“心情 ID”组成：

```text
{costumeId}_{moodId}.png
```

例如：

```text
summer_home_happy.png
```

含义：

```text
summer_home = 服装 ID
happy = 心情 ID
```

这张图会在后台选择“服装：summer_home”和“心情：happy”时自动显示。

推荐完整路径：

```text
assets/packs/{packId}/avatars/{costumeId}_{moodId}.png
```

例如：

```text
assets/packs/summer-pack/avatars/summer_home_happy.png
```

路径含义：

```text
summer-pack             扩展包文件夹
avatars                 立绘文件夹
summer_home_happy.png   夏日居家服装 + 开心状态
```

如果新服装 `summer_home` 要支持默认 6 个心情，需要准备：

```text
summer_home_idle.png
summer_home_happy.png
summer_home_shy.png
summer_home_care.png
summer_home_encourage.png
summer_home_sleepy.png
```

默认心情对应关系：

```text
idle       待机
happy      开心
shy        害羞
care       关心
encourage  鼓励
sleepy     困倦
```

如果后续新增心情，例如：

```text
surprised  惊喜
```

需要增加对应立绘：

```text
summer_home_surprised.png
```

命名建议：

- `costumeId` 和 `moodId` 使用英文小写、数字、下划线。
- 不建议使用中文、空格或特殊符号。
- 文件名需要和 `manifest.json` 中的服装 ID、心情 ID 保持一致。
- 本地回复主要自动切换心情；只有明确换装请求，或首次进入清晰新场景时，才会做保守的服装切换。
- 同一场景持续对话时不应反复换装，只切换心情；切换到新场景或明确要求换装时才允许再次换服装。
- “好累啊、困了、晚安、想睡”等表达只代表需要关心或困倦心情，不应单独触发居家服装；只有“回家、居家、休息模式、准备睡觉、躺一会”等更明确场景才可考虑居家服装。

## 大模型接入注意事项

- 后台“模型接口”页已预留 OpenAI、MiniMax、DeepSeek、通义千问、智谱 GLM、Kimi、硅基流动、Ollama、LM Studio 和自定义兼容接口。
- 后台首页背景故事只用于展示和产品氛围，不作为正式人格内容，也不会注入大模型提示词。
- 后台“角色人格”页的角色名称、自称、用户称呼会优先注入大模型提示词；默认分别为“桌面陪伴精灵妹妹”“妹妹”“哥哥”。
- 远程厂商默认按 OpenAI-compatible 聊天补全格式请求；多数会规范到 `/v1/chat/completions`，DeepSeek 会规范到 `/chat/completions`。如果厂商控制台给出的地址不同，以控制台官方文档为准，直接覆盖“接口链接”或“标准端点”即可。
- API Key 会保存在本地 `data/model-secrets.json`，不要提交、分享或打包给别人。
- 模型回复建议返回 JSON：

```json
{
  "text": "显示在气泡里的中文回复",
  "mood": "happy",
  "costumeId": ""
}
```

- `mood` 必须是后台素材里存在的心情 id，程序会据此自动切换对应心情立绘。
- `costumeId` 只有在明确需要换装、切换风格，或首次进入清晰新场景时填写；普通聊天和同一场景延续对话应留空，避免模型自动乱换风格。
- 程序会校验模型返回的 `mood` 和 `costumeId`，无效值会被忽略并回退到当前状态。

## 回复优先级与兜底

- 启用大模型且配置有效时，普通聊天始终优先请求大模型。
- 大模型暂时无法回复时，程序会自动检索本地回复库。
- 本地回复检索顺序为：日记本关键点问答 -> 回复规则关键词 -> `fallback` 兜底回复。
- 本地回复可以绑定心情；明确换装语境，或首次进入工作、护理、运动、学习、旅行等清晰新场景时，也可以做保守的服装兜底推断。
- 本地关键点问答、关键词回复和 fallback 兜底都必须经过同一套场景守则，不允许任何本地回复绕过该规则直接换装。
- 本地遇到“好累啊”这类短句时，应优先命中 `comfort` / `care` 一类心情回复，不切换服装。
- 本地回复只作为兜底，不应该抢在可用的大模型前面回答。
- 兜底文案应保持妹妹感和陪伴感，不要直接把“模型失效、接口失败”等技术细节显示给用户。

## 服装场景说明

- `data/assets.json` 中每套服装可以配置 `modelHint`。
- `modelHint` 只给大模型作为判断参考，不是强制优先级。
- 新增服装时建议写清楚适用场景，例如工作、学习、旅行、运动、护理、正式场合、日常外出等。
